**Business Case ตั้งต้น**

*   **ชื่อ Case:** "ค้นหากลุ่มลูกค้าเป้าหมายสำหรับแคมเปญบัตรเครดิต Co-brand ร้านอาหาร"
*   **ผู้ใช้ (Persona):** เจ้าหน้าที่การตลาด (Marketing Officer)
*   **สถานการณ์:** ธนาคารกำลังจะเปิดตัวบัตรเครดิต Co-brand ใหม่ร่วมกับเครือร้านอาหารชื่อดัง "Gourmet Eats" และต้องการหากลุ่มลูกค้าปัจจุบันที่น่าจะสนใจบัตรใบนี้มากที่สุด เพื่อส่งข้อเสนอพิเศษสำหรับการสมัคร
*   **เป้าหมายของเจ้าหน้าที่การตลาด:**
    *   ระบุลูกค้าที่ใช้จ่ายในหมวดหมู่ "ร้านอาหาร" (Dining MCCs) เป็นจำนวนมากในช่วง 3 เดือนที่ผ่านมา
    *   แต่ยังไม่ได้ถือบัตรเครดิต Co-brand อื่นๆ ของธนาคาร (เพื่อลดความซ้ำซ้อนและเพิ่มโอกาสในการสมัครบัตรใหม่)
    *   และอยู่ในกลุ่มลูกค้าที่มีศักยภาพ (เช่น ไม่ใช่กลุ่มลูกค้าที่มีความเสี่ยงสูง หรือกลุ่มที่เพิ่งยกเลิกบัตรไป)
*   **สิ่งที่ระบบ (Backend) ต้องทำ:**
    1.  รับ Input จาก Frontend (ที่มาจาก Wireframe):
        *   หมวดหมู่ร้านค้าที่สนใจ (เช่น "Dining")
        *   ช่วงเวลา (เช่น "3 เดือนล่าสุด")
        *   ยอดใช้จ่ายขั้นต่ำในหมวดหมู่นั้น (เช่น มากกว่า 5,000 บาท)
        *   (Optional) กลุ่มลูกค้าที่ต้องการ Target (เช่น "Gold Tier", "Platinum Tier")
        *   (Optional) รายชื่อบัตร Co-brand ที่ต้องการคัดกรองออก (Exclude list)
    2.  ประมวลผลข้อมูลลูกค้าและข้อมูลธุรกรรมบัตรเครดิต
    3.  ส่งคืนรายชื่อลูกค้า (Customer ID, ชื่อ, ยอดใช้จ่ายในหมวดหมู่, Segment) ที่เข้าเกณฑ์

**BRD Items ที่เกี่ยวข้อง (แบบย่อ):**
*   BO1: ทำความเข้าใจรูปแบบการใช้จ่าย
*   BO2: ระบุ MCC ที่ใช้จ่ายบ่อย
*   BO4: เพิ่มประสิทธิภาพแคมเปญโปรโมชั่น
*   FR01: Data Ingestion (เบื้องหลัง, สมมติว่ามีข้อมูลแล้ว)
*   FR02: Customer Segmentation (อาจจะใช้ Segment ที่มีอยู่แล้ว หรือสมมติขึ้น)
*   FR03: Spending Pattern Analysis (เฉพาะส่วนที่เกี่ยวกับ MCC และยอดใช้จ่าย)
*   FR05: Promotion Candidate Identification (แกนหลักของ Case นี้)
*   FR06.2: Promotion Candidate List (ผลลัพธ์ที่ API จะส่งออก)
*   Data Requirements: Transaction Data, Customer Data, MCC Data.
