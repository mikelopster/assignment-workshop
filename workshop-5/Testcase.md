## Theme ของ Test Cases
"การค้นหาและกรองลูกค้าเป้าหมายสำหรับแคมเปญ Gourmet Eats Co-brand Card"

## Feature ที่มี
- การค้นหาและกรองลูกค้า: สามารถเลือกหมวดหมู่การใช้จ่าย (Dining, Travel, Shopping), เลือกช่วงเวลา (Last 3-12 Months), ระบุยอดใช้จ่ายขั้นต่ำ, เลือกกลุ่มลูกค้า (Gold, Platinum, Affluent) แบบเลือกได้หลายกลุ่ม, และยกเว้นบัตร Co-brand ที่ไม่ต้องการแบบเลือกได้หลายบัตร
- การแสดงผลการค้นหา: แสดงรายชื่อลูกค้าที่ตรงตามเงื่อนไข พร้อมยอดใช้จ่ายรวมและ Segment ของลูกค้า, แสดงข้อความเมื่อไม่พบลูกค้าที่ตรงตามเงื่อนไข
- การตรวจสอบความถูกต้อง: ตรวจสอบข้อมูลนำเข้า (ห้ามค่าติดลบ, ต้องเป็นตัวเลข), คำนวณยอดใช้จ่ายรวมจากธุรกรรมในช่วงเวลาที่กำหนด, กรองธุรกรรมตามช่วงเวลาอย่างถูกต้อง
- ค่าเริ่มต้นระบบ: มีการตั้งค่าเริ่มต้นสำหรับหมวดหมู่ (Dining), ช่วงเวลา (Last 3 Months), กลุ่มลูกค้า (All Segments) และไม่มีการยกเว้นบัตรใดๆ
- การจัดการกรณีพิเศษ: รองรับอักขระพิเศษในชื่อลูกค้า, ตัดช่องว่างนำหน้าและต่อท้าย, ไม่คำนึงถึงตัวพิมพ์ใหญ่-เล็ก, จัดการข้อมูล null หรือว่างเปล่า
- ประสิทธิภาพและความสอดคล้อง: แสดงผลการค้นหาภายในเวลาที่กำหนด, แสดงผลลัพธ์ที่สอดคล้องกันเมื่อใช้เงื่อนไขเดียวกัน, มีโครงสร้าง API ที่ชัดเจนทั้งกรณีสำเร็จและไม่สำเร็จ

## สมมติฐานสำหรับ Test Cases:
- มีข้อมูลลูกค้า (Customer Data) และข้อมูลธุรกรรม (Transaction Data) เพียงพอในระบบทดสอบ
- มีข้อมูล Merchant Category Code (MCC) ที่ถูกต้องสำหรับการระบุหมวดหมู่ "Dining" (ร้านอาหาร) และหมวดหมู่อื่นๆ
- มีข้อมูล Segment ของลูกค้า (เช่น Gold, Platinum)
- มีข้อมูลว่าลูกค้าคนไหนถือบัตร Co-brand อื่นๆ อยู่บ้าง
